# README 中文化与安全声明 Design

## 背景

当前仓库仅有英文 `README.md`。用户需要：

1. 提供独立简体中文版 README。
2. 在英文 README 中加入“简体中文”入口。
3. 强调用户必须手动执行两条初始化命令，并说明按提示输入。
4. 说明顽鹿密码输入为隐藏模式（命令行不回显）。
5. 增加使用与安全免责声明（学习用途、非商用、本地保存、勿向 agent 提供账号信息、后果自负）。

## 设计决策

- 采用双文档方案：保留 `README.md` 作为英文主文档，新增 `README.zh-CN.md` 作为完整中文版。
- 在 `README.md` 顶部添加 `简体中文` 链接，降低中文用户进入成本。
- 在两份 README 中都加入初始化说明和免责声明，避免只看单一语言文档时遗漏关键信息。

## 结构方案

### `README.md`

- 标题后新增语言链接：`[简体中文](README.zh-CN.md)`
- 新增初始化提示段：
  - `python scripts/bootstrap.py --onelap-auth-init`
  - `python scripts/bootstrap.py --strava-auth-init`
  - 按提示输入
  - OneLap/顽鹿密码输入不回显
- 新增免责声明段（英文）

### `README.zh-CN.md`

- 与英文 README 主体结构保持一致
- 单独“初始化（需手动执行）”小节，包含两条命令与隐藏输入提示
- 单独“免责声明”小节，完整覆盖用户指定声明

## 验收标准

1. `README.md` 包含并可点击跳转到 `README.zh-CN.md` 的“简体中文”链接。
2. `README.zh-CN.md` 存在且内容完整可读。
3. 两条初始化命令在文档中完整、无改写、可直接复制执行。
4. 明确写明顽鹿密码输入不回显属于正常现象。
5. 免责声明覆盖学习用途、非商用、本地保存、不要向 agent 提供账号信息、后果自负。
